# Task 1.3: API Server Foundation

**Phase**: 1 - Project Setup & Infrastructure
**Status**: Not Started

## Summary

Set up Node.js API server with authentication and core middleware.

## Why This Matters

Foundation for all backend functionality - needs robust auth, error handling, and clean routing.

## Inputs/Outputs

- **Input**: Schema, auth requirements
- **Output**: Running API server with auth endpoints

## Technical Notes

- Use Fastify (faster) or Express (more familiar)
- OAuth implementation via Passport.js or custom
- Support Discord, GitHub, Google providers
- JWT sessions with refresh tokens
- Rate limiting on public endpoints
- CORS configured for frontend origin

## Files to Touch

- `packages/api/src/index.ts`
- `packages/api/src/routes/*`
- `packages/api/src/middleware/*`
- `packages/api/src/auth/*`

## Acceptance Criteria

- [ ] Server starts and responds to health check
- [ ] OAuth login flow works (at least one provider)
- [ ] JWT auth middleware protects routes
- [ ] Error handling returns consistent format
